### How to Generate ABI

**Option 1: Use slash command (easiest)**
```bash
/generate-abi <CONTRACT_ADDRESS> <CHAIN>
# Example: /generate-abi 0xBBBBBbbBBb9cC5e90e3b3Af64bdAF62C37EEFFCb ethereum
```

**Option 2: Use npx directly**
```bash
npx @subsquid/evm-typegen <CONTRACT_ADDRESS>#<EVENT_NAME> --chain <CHAIN>
```

### What Gets Generated

The generated ABI includes:
- Correct keccak256 topic hashes
- Proper `event()` function with `@subsquid/evm-abi`
- Typed parameters using `@subsquid/evm-codec`
- Correct event structure (evt.block.number, evt.timestamp, etc.)

### NEVER Do This

**DO NOT manually create ABI files like this:**
```typescript
// BAD - Will cause "undefined" topic errors
export const events = {
  Transfer: { /* manually typed */ }
}
```

**ALWAYS use generated ABIs:**
```typescript
// GOOD - Generated by evm-typegen
import * as p from '@subsquid/evm-codec'
import { event, indexed } from '@subsquid/evm-abi'

export const Transfer = event(
  '0xddf252ad1be2c89b69c2b068fc378daa952ba7f163c4a11628f55a4df523b3ef',
  'Transfer(address,address,uint256)',
  { from: indexed(p.address), to: indexed(p.address), value: p.uint256 }
)
```

**This is Pipes SDK 101 - skipping this step will cause cascading errors.**

### Step 1: Generate ABI (30 seconds) - MANDATORY

Now that you have the contract address, generate TypeScript types:

```bash
/generate-abi <CONTRACT_ADDRESS> <CHAIN>
# Example: /generate-abi 0x4e65fE4DbA92790696d040ac24Aa414708F5c0AB base
```

**Validation Checklist** (all must pass):
- [ ] ABI file exists at `src/abi/<contract>.ts`
- [ ] File contains `export const events = { ... }` or individual event exports
- [ ] Topic hashes are 66-character hex strings (0x + 64 chars)
- [ ] Event parameter names match what you'll use in code

**If validation fails**: Do NOT proceed. Fix ABI generation first.

**Proxy Contract Warning**: If your contract is a proxy (like Lido stETH), the ABI generator may only get the proxy ABI, not the implementation events. In this case:
1. Use `commonAbis` if available (e.g., `commonAbis.erc20.events.Transfer`)
2. Or define events inline - see "Lessons Learned" section below

**Action**: Verify ABI was generated
```bash

grep "export const" src/abi/*.ts | head -5
```

First, check what's available:
```bash
node -e "import('@subsquid/pipes/evm').then(m => console.log(Object.keys(m.commonAbis)))"
# Output: [ 'erc20' ]  ← Only erc20 is available
```

If your events aren't in commonAbis, create a custom ABI file:

1. Create `src/abi/yourContract.ts`:
```typescript
import { event, indexed } from '@subsquid/evm-abi'
import * as p from '@subsquid/evm-codec'

export const events = {
  Deposit: event(
    '0xe1fffcc4923d04b559f4d29a8bfc6cda04eb5b0d3c460751c2402c5c5cc9109c',
    'Deposit(address,uint256)',
    { dst: indexed(p.address), wad: p.uint256 },
  ),
  Withdrawal: event(
    '0x7fcf532c15f0a6db0bd6d0e038bea71d30d808c7d98cb3bf7268a95bf5081b65',
    'Withdrawal(address,uint256)',
    { src: indexed(p.address), wad: p.uint256 },
  ),
}
```

2. Import in your main file:
```typescript
import * as myContractAbi from './abi/yourContract.js'

events: {
  deposits: myContractAbi.events.Deposit,
  withdrawals: myContractAbi.events.Withdrawal,
}
```

**Finding event signatures and topics:**
- Use Etherscan/Basescan contract page → "Events" tab
- Or use `/generate-abi` command (see .claude/commands/generate-abi.md)

Make sure the key (e.g., `transfers`) is:
- Plural (transfers not transfer)
- Descriptive
- Matches what you use in `.pipe()`

#### 6e. Update Data Transformation
```typescript
.pipe(({ transfers }) => {  // Key matches events above
  return transfers.map((t) => ({
    blockNumber: t.block.number,
    txHash: t.rawEvent.transactionHash,
    value: t.event.value.toString(),  // BigInt to string!
  }))
})
```

#### 6f. Update Table Schema and Inserts
```typescript
onStart: async ({ store }) => {
  await store.client.command({
    query: `
      CREATE TABLE IF NOT EXISTS your_table (
        block_number UInt32,
        tx_hash String,
        value String
      ) ENGINE = MergeTree() ORDER BY block_number
    `,
  })
}

onData: async ({ data, store }) => {
  if (data.length > 0) {
    await store.client.insert({
      table: "your_table",  // Match table name above
      values: data,
      format: "JSONEachRow",
    })
  }
}
```

### After 1st Error

1. **Compare to working example**
   - Open example side-by-side
   - Find what's different
   - Fix the difference

2. **Common Error #1 fixes**:
   | Error Message | Fix |
   |---------------|-----|
   | `Package subpath './clickhouse' is not defined` | Change to `@subsquid/pipes/targets/clickhouse` |
   | `Export named 'createClickHouseTarget' not found` | Use `clickhouseTarget` |
   | `Cannot read properties of undefined (reading 'map')` | Check event key matches between `evmDecoder.events` and `.pipe()` |
   | `Authentication failed` | Add `username: "default", password: "default"` |

3. **Try fix once**

**STOP CODING IMMEDIATELY**

1. Create TodoWrite entry:
   ```
   STOP - Research API for [specific component that's failing]
   ```

   - Read line by line
   - Don't skim


5. Document what you learned

6. Try ONCE more with correct pattern

## Quick Reference: Common Patterns

### Pattern: Basic EVM Event Tracking

```typescript
await evmPortalSource({
  portal: "https://portal.sqd.dev/datasets/ethereum-mainnet",
})
  .pipe(
    evmDecoder({
      range: { from: "21000000" },
      contracts: ["0xAddress"],
      events: {
        transfers: commonAbis.erc20.events.Transfer,
      },
    })
  )
  .pipe(({ transfers }) =>
    transfers.map((t) => ({
      txHash: t.rawEvent.transactionHash,
      from: t.event.from,
      to: t.event.to,
      value: t.event.value.toString(),
    }))
  )
  .pipeTo(clickhouseTarget({ /* ... */ }))
```

### Pattern: Factory-Based Contract Discovery

```typescript
import { factory, factorySqliteDatabase } from "@subsquid/pipes/evm"

evmDecoder({
  range: { from: "12369621" },
  contracts: factory({
    address: "0xFactoryAddress",
    event: factoryAbi.events.PairCreated,
    parameter: "pair",  // Which event parameter contains the new contract address
    database: factorySqliteDatabase({ path: "./pools.sqlite" }),
  }),
  events: {
    swaps: pairAbi.events.Swap,
  },
})
```

## Lessons Learned (From Real Testing)

These issues were discovered through actual template testing and indexer creation:

### Network Name Mapping
Portal expects specific network names that don't always match common usage:
- `eth-mainnet` → `ethereum-mainnet`
- `arbitrum-mainnet` → `arbitrum-one`
- `zksync-era` → `zksync-mainnet`
- `bsc-mainnet` → `binance-mainnet`

**Quick validation**:
```bash
curl -s -o /dev/null -w "%{http_code}" https://portal.sqd.dev/datasets/<chain-name>
# 200 = valid, 404 = wrong name
```

### ClickHouse Password Mismatch
Generated `.env` files often have `CLICKHOUSE_PASSWORD=password` but local Docker uses `default`:
```bash
# Fix in .env file
CLICKHOUSE_PASSWORD=default
```

### Proxy Contracts (Lido, Upgradeable Contracts)
ABI generation for proxy contracts only gets the proxy ABI, not implementation events.

**Solution**: Use inline event definitions:
```typescript
import { event, indexed } from '@subsquid/evm-abi'
import * as p from '@subsquid/evm-codec'

// Define event inline instead of relying on ABI generation
const Submitted = event(
  '0x96a25c8ce0baabc1fdefd93e9ed25d8e092a3c2c1e96a9a2a6f3b4e9e8e0c7f3',
  'Submitted(address,uint256,address)',
  { sender: indexed(p.address), amount: p.uint256, referral: p.address },
)
```

### Event Field Name Mismatches
Different protocols use different field names for similar concepts:
- Morpho vaults: `onBehalf` (not `owner`)
- ERC4626 standard: `owner`
- Some vaults: `receiver`

**Always check the actual ABI** before assuming field names.

### Shared Checkpoint IDs
All indexers using default `settings: { id: "stream" }` share the same checkpoint, causing resume issues.

**Solution**: Use unique IDs per indexer:
```typescript
settings: { id: "morpho-deposits-base" }  // Unique per project
```

### ABI File Naming
Generated ABI files are named by contract address (e.g., `0xBBBB...FCb.ts`). You may want to rename for clarity:
```bash
mv src/abi/0xBBBBBbbBBb9cC5e90e3b3Af64bdAF62C37EEFFCb.ts src/abi/morpho-blue.ts
```
